# 加密貨幣 AI 交易策略開發說明書
> 整合 Mark Minervini、Qullamaggie、Martin Schwartz 核心邏輯，針對加密貨幣市場（24/7、高波動、合約槓桿）進行調整

---

## 核心差異：加密貨幣 vs 傳統股市

| 指標名稱 | 傳統股市用法 | 加密貨幣 AI 調整用法 |
|---------|------------|------------------|
| 均線 (MA) | 50/200 日線判定趨勢 | 4H EMA 200 判定趨勢；1H EMA 10/20 追蹤動能 |
| 成交量 (VOL) | 僅看現貨成交量 | 結合持倉量 (OI) 與清算數據 (Liquidation) |
| 形態 (VCP) | 收縮區間通常很窄（5%） | 收縮比例放寬至 10-15%，結合 0.618 斐波那契位 |
| RSI / MACD | 判斷超買超賣 | 增加「資金費率」過濾，防止多頭擁擠爆倉 |
| 止損規則 | 固定百分比（4-8%） | ATR 動態止損，防止被隨機插針掃損 |

## 模組一：均線系統（Crypto 調整版）

### 核心邏輯
加密貨幣趨勢演變極快，200 日均線反應太慢，需縮短時間框架。

### 參數設定
- **大趨勢過濾**：4H EMA 200 為牛熊分界線
  - 現價在 EMA 200 上方 → 只做多
  - 現價在 EMA 200 下方 → 只做空或觀望
- **動能追蹤**：1H EMA 10 / EMA 20
  - EMA 10 向上交叉 EMA 20 → 做多信號
  - EMA 10 向下交叉 EMA 20 → 做空信號

### AI 執行規則
```
進場條件：現價 > 4H EMA 200 且 1H EMA 10 向上交叉 EMA 20
出場條件：1H 收盤跌破 EMA 20，立即判定短期動能衰竭，執行平倉
強勢幣種：沿 15 分鐘或 1H EMA 10 運行時持續持倉
```

## 模組二：成交量 + 持倉量（OI）驗證

### 核心邏輯
加密貨幣成交量需結合合約市場的持倉量（Open Interest）才能判斷真假突破。

### 判斷規則

| 情況 | 成交量 | OI 變化 | 判斷 | AI 動作 |
|------|-------|---------|------|---------|
| 真突破 | 放量（2倍均量以上） | 增加 | 機構真實進場 | 確認進場 |
| 假突破 | 放量 | 減少 | 空頭爆倉（Short Squeeze） | 禁止追多 |
| 洗盤 | 量縮 | 穩定 | 無大資金出逃 | 繼續持倉 |
| 出逃 | 天量長陰 | 大幅減少 | 機構撤退 | 立即平倉 |

### AI 執行規則
```
突破驗證：成交量 ≥ 前 5-10 根 K 線均量的 2 倍
OI 確認：突破同時 OI 增加 → 真突破
風險規避：多空爆倉比出現極端值時，暫停突破策略（防止插針）
```

## 模組三：MACD + RSI 共振系統

### MACD 核心邏輯（指標之王）

| 信號 | 條件 | AI 動作 |
|------|------|---------|
| 最強做多 | 零軸上方金叉 + RSI > 50 | 重倉進場 |
| 一般做多 | 金叉但在零軸下方 | 輕倉試水 |
| 頂背離警告 | 價格創新高但 MACD 高點降低 | 啟動移動止損 |
| 底背離機會 | 價格創新低但 MACD 低點抬高 | 標記潛在反彈 |
| 出場信號 | MACD 柱線縮短或死叉 | 分批止盈 |

### RSI 核心邏輯

| RSI 數值 | 狀態 | AI 動作 |
|---------|------|---------|
| > 85 | 極端超買 | **禁止追多** |
| 70 ~ 85 | 超買偏熱 | 謹慎，設移動止損 |
| 50 ~ 70 | 多頭強勢 | 可進場，健康區間 |
| 30 ~ 50 | 中性偏弱 | 觀望，等待確認 |
| < 30 | 超賣 | 標記潛在反彈區域 |

### AI 執行規則
```
進場：MACD 金叉 + 柱線轉正 + RSI 守在 50 以上
最強進場：MACD 在零軸上方金叉（勝率最高）
禁止進場：RSI > 85（極端超買）
出場參考：RSI 出現頂背離，啟動移動止損鎖利
```

## 模組四：資金費率（Funding Rate）過濾器

### 核心邏輯
資金費率是加密貨幣特有指標，反映市場情緒過熱程度，是防止「多殺多」的關鍵過濾器。

### 判斷標準

| 資金費率 | 市場狀態 | AI 動作 |
|---------|---------|---------|
| > +0.03% / 8小時 | 多頭過度擁擠 | **禁止做多** |
| +0.01% ~ +0.03% | 偏熱，需謹慎 | 輕倉，設緊止損 |
| -0.01% ~ +0.01% | 情緒中性 | 正常操作 |
| < -0.01% | 空頭佔優 | 觀察反彈機會 |
| 極端負值（< -0.05%） | 空頭過度擁擠 | 可能底部反轉信號 |

### AI 執行規則
```
進場前必查：資金費率是否低於 0.03%（每 8 小時）
禁止做多：費率超過 0.03%，即便 MACD 金叉、RSI 強勢也禁止追多
逆向信號：價格下跌且費率極端負值 → 標記為潛在底部反轉
數據來源：OKX 資金費率 API / Coinglass / Velo
```

## 模組五：ATR 動態止損系統

### 核心邏輯
加密貨幣隨機插針（Wick）頻繁，固定百分比止損極易被掃損，必須使用動態止損。

### 計算公式
```
止損位 = 進場價 - (2 × ATR)
止盈位 = 進場價 + (4 × ATR)  → 1:2 盈虧比
止盈位 = 進場價 + (6 × ATR)  → 1:3 盈虧比（最理想）
```

### 動態調整規則

| 市場狀態 | ATR 倍數 | 說明 |
|---------|---------|------|
| 高波動時段（美股開盤、重大數據） | 2.5 ~ 3× | 自動放寬，防止插針 |
| 正常交易時段 | 2× | 標準設置 |
| 震盪盤整 | 1.5× | 縮小止損，提高資金效率 |

### 盈虧比規則
```
最低要求：盈虧比 1:2
理想目標：盈虧比 1:3
絕對禁止：盈虧比低於 1:1.5 的交易
```

## 綜合進場條件：七項共振檢查清單

> AI 應在以下條件盡量滿足時才執行交易，條件越多越接近重倉

| 序號 | 條件 | 描述 | 重要性 |
|------|------|------|--------|
| ✅ 1 | 大趨勢確認 | 現價在 4H EMA 200 上方 | 必要條件 |
| ✅ 2 | 動能啟動 | 1H EMA 10 向上交叉 EMA 20 | 必要條件 |
| ✅ 3 | MACD 共振 | 金叉且柱線轉正（零軸上方最佳） | 必要條件 |
| ✅ 4 | RSI 強勢 | RSI 守在 50 以上且未超過 85 | 必要條件 |
| ✅ 5 | 量能確認 | 放量突破 + OI 同步增加 | 高權重 |
| ✅ 6 | 費率正常 | 資金費率低於 0.03% / 8小時 | 高權重 |
| ✅ 7 | 止損設置 | ATR 動態止損已設定，盈虧比 ≥ 1:2 | 必要條件 |

### 倉位管理對應表

| 共振分數 | 建議倉位 | 說明 |
|---------|---------|------|
| 0 ~ 2 項 | 0%（禁止進場） | 條件不足，等待 |
| 3 ~ 4 項 | 30% 輕倉 | 試水，嚴格止損 |
| 5 項 | 50% 半倉 | 可進場，標準止損 |
| 6 項 | 70% | 條件優良 |
| 7 項全部 | 100% 重倉 | 最佳進場時機 |

## 特殊情況：禁止進場條件

以下任一條件成立，AI 應**強制禁止進場**：

```
❌ 資金費率 > 0.03% / 8小時（多頭過熱）
❌ RSI > 85（極端超買）
❌ 放量突破但 OI 下降（假突破）
❌ 現價在 4H EMA 200 下方（空頭趨勢）
❌ MACD 出現頂背離（趨勢反轉警告）
❌ 多空爆倉比出現極端值（市場即將插針）
❌ 盈虧比低於 1:1.5
```

## VCP 形態（加密貨幣調整版）

### 原始 VCP vs 加密貨幣 VCP

| 項目 | 傳統股市 | 加密貨幣調整 |
|------|---------|------------|
| 收縮次數 | 2 ~ 4 次 | 2 ~ 3 次（波動快，次數較少） |
| 收縮幅度 | 5% ~ 10% | 10% ~ 15%（放寬） |
| 支撐確認 | 技術形態 | 0.618 斐波那契位 + 委託單密集區 |
| 成交量 | 萎縮確認 | 萎縮 + OI 穩定 |

### 進場條件
```
1. 價格完成 2-3 次波動收縮，幅度遞減
2. 最後一次收縮落在 0.618 斐波那契支撐位附近
3. 委託單簿顯示該價位有大量掛單支撐
4. 等待放量突破緊湊區上軌
5. 突破時 OI 同步增加確認真突破
```

## 風險管理原則

### 單筆交易風險
```
每筆交易最大虧損：不超過總資金的 2%
單日最大虧損上限：總資金的 5%
觸及日虧損上限：立即停止交易，次日重新評估
```

### 槓桿使用建議

| 市場狀態 | 建議槓桿 |
|---------|---------|
| 七項共振全部成立 | 最高 10x |
| 5 ~ 6 項共振 | 最高 5x |
| 3 ~ 4 項共振 | 最高 3x |
| 市場高波動期 | 最高 2x |

### 心理紀律規則
```
虧損時：禁止加倉攤平（除非重新滿足進場條件）
獲利時：價格達目標 50% 時先出場一半，鎖定部分利潤
連續虧損：連虧 3 筆後強制休息，檢視策略
```

## AI 開發指令（技術實作建議）

```
時間框架：以 1 小時為主要操作級別，4 小時為趨勢過濾

進場判斷邏輯（需同時滿足）：
  IF 現價 > 4H_EMA_200
  AND 1H_EMA_10 上穿 1H_EMA_20
  AND MACD_柱線 > 0
  AND RSI > 50 AND RSI < 85
  AND 成交量 > 均量_5日 × 2
  AND OI_變化 > 0
  AND 資金費率 < 0.03%
  THEN 允許進場，設置止損 = 進場價 - 2×ATR

出場邏輯：
  IF 1H 收盤 < 1H_EMA_20
  OR MACD 死叉
  OR 觸及止盈價
  OR 觸及止損價
  THEN 執行平倉

風險過濾：
  IF 資金費率 > 0.03% THEN 禁止做多
  IF RSI > 85 THEN 禁止做多
  IF OI 下降 AND 成交量放大 THEN 標記假突破警告
```
